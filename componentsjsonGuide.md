# Guide: components.json

## What is this file?
`components.json` is the **configuration file for shadcn/ui** — the component library this project uses. shadcn/ui is different from most libraries: instead of installing it as an npm package, you use a CLI tool that copies component source code directly into your project. This file tells the CLI where to put things.

## What does it control?

| Setting | Value | What it means |
|---|---|---|
| `$schema` | `https://ui.shadcn.com/schema.json` | Links to the official schema for validation/autocomplete in your editor |
| `style` | `"new-york"` | The visual style of components. "new-york" is the newer, tighter style. The alternative is "default" (older, more spaced). |
| `rsc` | `true` | React Server Components enabled. This is required for Next.js App Router. |
| `tsx` | `true` | Components are written in TypeScript (`.tsx` files), not plain JavaScript. |
| `tailwind.config` | `""` | Empty because Tailwind v4 doesn't use a config file path (it uses CSS-based config). |
| `tailwind.css` | `"src/app/globals.css"` | The path to your main CSS file where Tailwind variables live. |
| `tailwind.baseColor` | `"neutral"` | The base color palette (neutral gray). Components use this as the default. |
| `tailwind.cssVariables` | `true` | Use CSS variables for colors (allows dark mode, theming) instead of hardcoded Tailwind color classes. |
| `tailwind.prefix` | `""` | No prefix on generated class names. |
| `aliases.components` | `"@/components"` | When the CLI adds a new component, it imports from this path. |
| `aliases.utils` | `"@/lib/utils"` | Path to the `cn()` helper function. |
| `aliases.ui` | `"@/components/ui"` | Where shadcn/ui components get saved (this is why all UI components are in `src/components/ui/`). |
| `aliases.lib` | `"@/lib"` | Path to your lib folder. |
| `aliases.hooks` | `"@/hooks"` | Path to your hooks folder. |
| `iconLibrary` | `"lucide"` | Which icon library to use. Lucide is the default for shadcn/ui. |

## When would you edit this file?
**Almost never.** This file is set up once when you initialize shadcn/ui and then left alone. You'd only edit it if you:
- Changed your folder structure (e.g. moved `components/` to `src/components/`)
- Switched icon libraries
- Changed your base color

## How to use it
You don't run this file directly. When you want to add a new shadcn/ui component, you run:
```bash
npx shadcn@latest add <component-name>
```
The CLI reads `components.json` to know where to save the component file and how to set up imports.

## The 40+ files in src/components/ui/
All those files (button.tsx, input.tsx, select.tsx, accordion.tsx, etc.) were generated by the shadcn CLI using the settings in this file. You CAN edit them, but it's generally better not to — if you re-add a component later, your edits get overwritten.

## Common beginner questions

**Q: Do I need to touch this file to add a new component?**
A: No. Just run `npx shadcn@latest add <name>` and the CLI handles everything.

**Q: What's the difference between "new-york" and "default" style?**
A: "new-york" has tighter spacing and smaller components. "default" is more spread out. This project uses "new-york".

**Q: Why are there so many files in src/components/ui/?**
A: Because shadcn/ui copies component source code into your project (unlike other libraries that keep it in node_modules). This gives you full control to customize, but means you have 40+ files to maintain.
