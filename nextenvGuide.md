# Guide: next-env.d.ts

## What is this file?
`next-env.d.ts` is a **TypeScript declaration file** auto-generated by Next.js. It tells TypeScript about Next.js-specific types so your editor can provide autocomplete and type-checking.

## What's in it?
```typescript
/// <reference types="next" />
/// <reference types="next/image-types/global" />
import "./.next/dev/types/routes.d.ts";

// NOTE: This file should not be edited
// see https://nextjs.org/docs/app/api-reference/config/typescript for more information.
```

### Line by line:
- `/// <reference types="next" />` — Loads Next.js's TypeScript type definitions
- `/// <reference types="next/image-types/global" />` — Loads types for the `<Image>` component
- `import "./.next/dev/types/routes.d.ts"` — Loads auto-generated route types (so TypeScript knows about your page routes for type-safe navigation)
- `// NOTE: This file should not be edited` — Next.js overwrites this file on every build

## When would you edit this file?
**Never.** The file explicitly says "should not be edited." Next.js regenerates it automatically.

If you accidentally delete it, just run `bun run dev` or `bun run build` and Next.js will recreate it.

## Common beginner questions

**Q: Why is this file needed?**
A: Without it, TypeScript wouldn't know about Next.js types like `Metadata`, `NextConfig`, or the `<Image>` component's props. Your editor would show red squigglies everywhere.

**Q: What are `.d.ts` files?**
A: They're "declaration files" — they describe types without containing actual code. Think of them as a menu that tells TypeScript "these types exist" without providing the implementation. Libraries ship `.d.ts` files so TypeScript users get type-checking.

**Q: What is `routes.d.ts`?**
A: It's a file Next.js generates in `.next/dev/types/` that contains types for all your routes. This enables features like type-safe `<Link href="/about">` — TypeScript checks that `/about` is a real route.
